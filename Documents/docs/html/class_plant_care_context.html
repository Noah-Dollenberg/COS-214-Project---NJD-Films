Detailed Description

Context class for the Strategy pattern in plant care system.

This class serves as the Context in the Strategy design pattern, managing the relationship between a Plant and its care strategy. It allows dynamic switching between different plant care strategies at runtime, providing flexibility in how plants are cared for.

The PlantCareContext maintains references to both a Plant object and a PlantCareStrategy, delegating care operations to the current strategy while maintaining the plant's state and information.

Design Pattern Role:
  • Context: PlantCareContext (this class)
  • Strategy: PlantCareStrategy
  • ConcreteStrategy: FrequentWatering, ModerateWatering, MinimalWatering
Usage Example:
Plant* myPlant = new Rose();
PlantCareContext context(myPlant);
context.setStrategy(new ModerateWatering());
context.executeCare();

void PlantCareContext::executeCare ( )

Executes the current care strategy.

Delegates care operations to the current strategy

Calls the careForPlant() method on the current strategy if one is set. If no strategy is set (strategy is nullptr), no action is taken.

Precondition
Context should have a valid strategy set
Postcondition
Care instructions are executed according to current strategy
Note
No operation performed if strategy is nullptr
Warning
Ensure strategy is set before calling this method

Delegates plant care operations to the active strategy

This method implements the core functionality of the Strategy pattern by delegating the care operation to the currently set strategy. It includes a null check to prevent crashes when no strategy is set.

Algorithm:
  1. Check if strategy pointer is not nullptr
  2. If valid, call careForPlant() on the strategy
  3. If nullptr, perform no operation (silent failure)
Implementation Notes:
  • Uses defensive programming with null pointer check
  • Silent failure when no strategy is set
  • Delegates all care logic to strategy implementation
  • No direct plant interaction - strategy handles plant care

void PlantCareContext::setStrategy ( PlantCareStrategy * s)

Sets the care strategy for the plant.

Parameters
sPointer to the PlantCareStrategy to use

Changes the current care strategy to the provided strategy. This allows runtime switching between different care approaches based on plant needs or environmental conditions.

Precondition
s should be a valid PlantCareStrategy pointer
Postcondition
strategy member is updated to point to s
Note
Previous strategy is not deleted - caller retains ownership
Warning
Passing nullptr will disable care execution
Parameters
sPointer to the PlantCareStrategy implementation to use

This method implements the strategy setter functionality, allowing the context to switch between different care strategies at runtime. The method simply assigns the provided strategy pointer to the internal strategy member variable.

Implementation Notes:
  • No validation is performed on the input parameter
  • Previous strategy reference is overwritten without cleanup
  • Caller retains ownership of strategy objects
  • Supports nullptr assignment to disable strategy execution
